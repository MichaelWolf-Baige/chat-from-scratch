## 二、核心发现

### 发现 1：宽度优于深度（已验证）

在 ~115M 参数级别，统一使用 wiki_100M 数据：

```
Extreme Wide (d=896 L=12)  → PPL=18.75  ← 最优
Shallow-Wide (d=768 L=16)  → PPL=19.09
Extreme Deep (d=512 L=36)  → PPL=19.05
Deep-Thin    (d=576 L=30)  → PPL=19.48  ← 最差
```

**宽度 vs 深度差值：3.7%**（18.75 vs 19.48）

d=896 只有 12 层，d=576 有 30 层——层级差了 2.5 倍但 PPL 反而更差。

**机制解释**：
- SwiGLU FFN 容量 = `3 × d_model × d_ff`，宽度直接放大每层计算量
- 浅层模型梯度传播路径短，不易衰减
- 12 层已经足够学到有意义的表示

### 发现 2：数据规模持续有效

```
wiki_50M  → PPL=23.23  (基线)
wiki_100M → PPL=18.93  (-18.5%)  ← 最大跳变
wiki_200M → PPL=18.44  ( -2.6%)  ← 收益递减但未饱和
```

**关键结论**：100M 参数模型在 100M tokens 时**远未饱和**。
预计 wiki_400M 可到 PPL≈18.0-18.2，wiki_800M 可到 PPL≈17.6-17.9。

### 发现 3：加数据 > 加参数

```
C200 (99M, wiki_200M)     → PPL=18.44
Extreme Wide (119M, 100M) → PPL=18.75  ← +20%参数仅+1.7%PPL
Mid (188M, 100M)          → PPL=18.13  ← +58%参数仅+3.3%PPL vs EW
```

**参数效率**：C200（0.186 PPL/M参数）> Extreme Wide（0.158）> Mid（0.096）。

翻倍数据比加 20M 参数更有效。

### 发现 4：深宽平衡边界在 d=512-768

Extreme Deep (d=512 L=36, 19.05) ≈ Shallow-Wide (d=768 L=16, 19.09)
差值 0.04，在误差范围内。宽度和深度存在等价交换区间。

### 发现 5：窄而深是反模式

Deep-Thin (d=576 L=30, PPL=19.48) **比基线 (18.93) 更差**。
d_head=64 的窄表示瓶颈无法被 30 层深度补偿。
